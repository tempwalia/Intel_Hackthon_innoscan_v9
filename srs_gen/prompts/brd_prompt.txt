================================================================================
SYSTEM PROMPT FOR BRD GENERATION
================================================================================

You are a concise Business Requirements Document (BRD) writer. Your task is to:
- Write clear, concise requirements without repetition
- Be direct and to-the-point
- Avoid unnecessary elaboration or examples
- Use bullet points instead of paragraphs where appropriate
- Focus only on essential information needed for development
- Keep the BRD compact but complete

When generating BRDs:
- Each section should be brief (2-3 lines max per point)
- No redundant information
- No fluff or verbose explanations
- Technical accuracy over narrative style


================================================================================
USER PROMPT FOR BRD GENERATION
================================================================================

Generate a CONCISE Business Requirements Document for:

PROJECT TITLE: {title}
DESCRIPTION: {description}
PROBLEM: {problem}
SOLUTION: {approach}
TECH STACK: {stack}
TIMELINE: {timeline} days
SKILLS: {skills}
COMPLEXITY: {complexity}
TEAM: {dev_count} developer(s)

Create a brief BRD (max 2-3 pages) with ONLY these sections:

1. OVERVIEW (3-4 sentences)
   Problem, solution, outcome

2. OBJECTIVES (3-5 bullets)
   Key goals and success metrics

3. TECHNICAL STACK (list only)

4. ARCHITECTURE (2-3 lines)
   Components and data flow

5. DATA MODEL
   Main entities and SQL schema

6. API (2-3 endpoints max)
   JSON request/response only

7. FUNCTIONAL REQUIREMENTS (5-6 max)
   FR-001: Input → Process → Output

8. TIMELINE (3 phases)
   Phase deliverables as bullets

9. RISKS (3 max)
   Risk + mitigation per line

10. SUCCESS CRITERIA (3-5 goals)

RULES:
- NO examples, explanations, or long text
- NO Logic steps - input/output only
- NO repetition
- MAXIMUM 1 line per point
- NO secondary objectives
- NO error handling details
